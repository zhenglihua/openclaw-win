## 一、产品简介

OpenClaw Manager 是一款专为 Windows 用户设计的 OpenClaw AI 助手管理工具。它提供了图形化的界面，帮助用户完成 OpenClaw 的安装、配置、监控和管理工作。

### 1.1 主要功能

- **一键安装**: 自动检测环境并安装 OpenClaw
- **状态监控**: 实时监控 Gateway 运行状态、端口占用、服务状态
- **服务管理**: 启动、停止、重启 OpenClaw Gateway
- **配置管理**: 自定义端口、API Key、Token 等参数
- **Windows 服务**: 支持将 OpenClaw 注册为 Windows 系统服务

### 1.2 系统要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10/11 (64位) |
| 运行环境 | .NET 8.0 (程序已包含) |
| Node.js | >= 22 (用于运行 OpenClaw) |
| 内存 | >= 4GB |
| 磁盘空间 | >= 200MB |

## 二、快速开始

### 2.1 首次使用

1. **下载安装包**: 获取 `OpenClaw-Win-v1.0.0-win-x64.zip`
2. **解压文件**: 解压到任意目录
3. **运行程序**: 双击 `OpenClawWin.exe`

### 2.2 首次运行向导

首次运行时，程序会引导您完成以下步骤：

1. **环境检测**: 检查 Node.js 和 OpenClaw 是否已安装
2. **安装 OpenClaw**: 如未安装，点击"开始安装"
3. **配置参数**: 设置 API Key、端口、Token 等
4. **保存配置**: 完成配置并启动服务

## 四、功能使用

### 4.1 仪表盘

仪表盘页面显示 OpenClaw 的整体状态，包含四个状态卡片：

| 状态项 | 说明 |
|--------|------|
| OpenClaw | 显示 OpenClaw 框架是否已安装 |
| 网关 | 显示 Gateway 进程是否运行 |
| 端口 | 显示端口 18789 的占用状态 |
| 服务 | 显示 Windows 服务注册状态 |

**快捷操作按钮**:
- **启动服务**: 启动 OpenClaw Gateway
- **停止服务**: 停止 Gateway 进程
- **重启服务**: 先停止再启动
- **打开Web UI**: 在浏览器中打开控制台
- **运行诊断**: 重新检测所有状态

### 4.2 安装向导

安装向导分 4 个步骤：

#### 步骤 1: 环境检测
- 自动检测 Node.js 是否已安装（需要 >= 22）
- 自动检测 OpenClaw 是否已安装
- 显示检测结果

#### 步骤 2: 安装 OpenClaw
- 点击"开始安装"按钮
- 程序执行 `npm install -g openclaw@latest`
- 显示安装进度和日志

#### 步骤 3: 配置参数
- **选择模型**: DeepSeek / GLM-4.7 / Qwen / OpenAI
- **API Key**: 输入对应的 API 密钥
- **端口**: 设置 Gateway 端口（默认 18789）
- **Token**: 设置 Web UI 访问令牌
- **工作区**: 设置工作目录路径

#### 步骤 4: 完成
- 保存配置到 `%APPDATA%\OpenClawWin\config.json`
- 确认安装完成

### 4.3 状态监控

状态监控页面提供实时状态更新：

**状态显示**:
- Gateway 进程状态（运行中/未运行）
- 端口占用状态（已占用/可用）
- Windows 服务状态（运行中/已停止/未注册）

**自动刷新**: 每 2 秒自动更新状态

**操作控制**:
- 启动/停止/重启服务
- 打开 Web 控制台
- 刷新日志
- 清空日志

**日志列表**: 显示最近 100 条操作记录

### 4.4 设置

设置页面包含以下选项：

| 设置项 | 说明 | 默认值 |
|--------|------|--------|
| 网关端口 | Gateway 监听端口 (1024-65535) | 18789 |
| 工作区路径 | OpenClaw 工作目录 | 用户目录 |
| 开机自启 | 开机自动启动程序 | 开启 |
| 启动最小化 | 启动时最小化到托盘 | 关闭 |
| 最小化到托盘 | 关闭时最小化到系统托盘 | 开启 |
| 自动检查更新 | 启动时检查更新 | 开启 |
| 日志级别 | Debug/Information/Warning/Error | Information |

**保存设置**: 点击"保存设置"按钮保存所有更改
**还原默认**: 点击"还原默认设置"恢复默认值

## 五、配置文件

### 5.1 配置文件位置

配置文件保存在: `%APPDATA%\OpenClawWin\config.json`

### 5.2 配置格式

```json
{
  "GatewayPort": 18789,
  "WorkspacePath": "",
  "AutoStartWithWindows": true,
  "AutoCheckUpdates": true,
  "LogLevel": "Information",
  "MinimizeToTray": true,
  "StartMinimized": false
}
```

### 5.3 日志文件

日志文件保存在: `%APPDATA%\OpenClawWin\logs\`

## 六、常见问题

### 6.1 安装问题

**Q: Node.js 检测失败**
> A: 请确保已安装 Node.js >= 22，并已将 node 添加到系统 PATH 环境变量

**Q: OpenClaw 安装失败**
> A: 检查网络连接是否正常，或尝试手动运行 `npm install -g openclaw@latest`

### 6.2 运行问题

**Q: 点击仪表盘后程序无响应**
> A: 已修复。请下载最新版本

**Q: 端口被占用**
> A: 关闭占用端口的应用程序，或在设置中更改 Gateway 端口

### 6.3 服务问题

**Q: Windows 服务无法启动**
> A: 请以管理员权限运行程序
